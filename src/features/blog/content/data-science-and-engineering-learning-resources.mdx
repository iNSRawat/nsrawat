## Part 1: Core Programming & Databases

The foundation of any data role lies in writing clean code and querying databases efficiently.

### 1. Master Python

Python is the undisputed king of data science and data engineering. From data wrangling to machine learning, Python's ecosystem is unparalleled.

```python
# Essential Python libraries for data
import numpy as np          # Numerical computing
import pandas as pd         # Data analysis & manipulation
import matplotlib.pyplot as plt  # Data visualization
```

- 🔗 **Start Learning:** [Master Python on Scrimba](https://scrimba.com/learn/python)

### 2. Learn SQL

SQL is the language of data retrieval. You must master joins, window functions, and query optimization to work with databases.

```sql
-- Example: Using a Common Table Expression (CTE) & Window Function
WITH RankedSales AS (
  SELECT
    employee_id,
    amount,
    ROW_NUMBER() OVER (PARTITION BY department_id ORDER BY amount DESC) as rank
  FROM sales
)
SELECT * FROM RankedSales WHERE rank = 1;
```

- 🔗 **Start Learning:** [Learn SQL on Codecademy](https://www.codecademy.com/learn/learn-sql)

### 3. Learn MySQL

MySQL is one of the most widely used relational database management systems (RDBMS) in the industry, powering millions of web applications and analytical layers.

## Part 2: Distributed Systems & Big Data

As datasets grow, single-node processing becomes a bottleneck. Distributed computing allows processing terabytes of data efficiently.

### 5. Dominate PySpark

PySpark combines the simplicity of Python with the power of Apache Spark, allowing you to perform distributed data processing at scale.

```python
# Initialize Spark Session and read a large dataset
from pyspark.sql import SparkSession

spark = SparkSession.builder \
    .appName("DataEngineeringRoadmap") \
    .getOrCreate()

df = spark.read.csv("s3://my-data-bucket/large_file.csv", header=True, inferSchema=True)
df.groupBy("category").count().show()
```

- 🔗 **Start Learning:** [Dominate PySpark on SparkByExamples](https://sparkbyexamples.com/pyspark-tutorial/)

## Part 3: Data Orchestration & Streaming

Data pipelines must run reliably, recover from failures, and handle real-time event streams.

### 6. Learn Bash, Airflow & Kafka

- **Bash:** Vital for command-line scripting, system automation, and Docker orchestration.
- **Apache Airflow:** The standard tool for scheduling and orchestrating complex workflows using Directed Acyclic Graphs (DAGs).
- **Apache Kafka:** A distributed event store and stream-processing platform for high-throughput, real-time data streams.

```python
# Simple Apache Airflow DAG Definition
from airflow import DAG
from airflow.operators.bash import BashOperator
from datetime import datetime

with DAG('daily_etl_pipeline', start_date=datetime(2026, 1, 1), schedule_interval='@daily') as dag:
    extract = BashOperator(task_id='extract_data', bash_command='python extract.py')
    transform = BashOperator(task_id='transform_data', bash_command='python transform.py')

    extract >> transform
```

- 🔗 **Start Learning:** [Learn Bash, Airflow & Kafka on edX](https://www.edx.org/course/building-etl-and-data-pipelines-with-bash-airflow-and-kafka)

### 10. Learn CI/CD Basics

Continuous Integration and Continuous Deployment (CI/CD) pipelines automate testing and deployment, reducing delivery risks.

```yaml
# Simple GitHub Actions workflow for testing code quality
name: Python Quality CI
on: [push]
jobs:
  build:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v3
      - name: Install dependencies
        run: pip install black pytest
      - name: Run Lint checks
        run: black --check .
      - name: Run Tests
        run: pytest
```

## Part 5: Modern Data Warehousing & Cloud Platforms

Modern analytical databases and lakehouses decouple storage and compute, enabling extremely fast analytical queries.

### 11. Decode Data Warehousing

Understand the core principles of OLAP databases, star/snowflake schemas, dimensional modeling (Fact vs. Dimension tables), and columnar-store architectures.

```sql
-- Star Schema: Joining Fact and Dimension tables
SELECT
    d.date_day,
    p.product_name,
    SUM(f.sales_amount) as total_sales
FROM fact_sales f
JOIN dim_date d ON f.date_key = d.date_key
JOIN dim_product p ON f.product_key = p.product_key
GROUP BY d.date_day, p.product_name;
```

- 🔗 **Start Learning:** [Decode Data Warehousing on Guru99](https://www.guru99.com/data-warehousing-tutorial.html)

### 12. Learn DBT (Data Build Tool)

DBT allows analytics engineers to transform data in their warehouse by simply writing SQL `SELECT` statements, bringing software engineering standards (like testing and version control) to analytics.

```sql
-- Example dbt model: stg_customers.sql
{{ config(materialized='view') }}

select
    id as customer_id,
    first_name,
    last_name
from {{ source('jaffle_shop', 'customers') }}
```

- 🔗 **Start Learning:** [Learn DBT on dbt Developer Hub](https://courses.getdbt.com/courses/fundamentals)

### 13. Learn Data Lakes

Learn how to store raw data in scalable object storage (like AWS S3 or Azure ADLS) using file formats like Parquet, ORC, and table formats like Delta Lake or Apache Iceberg.

```python
# Writing a pandas dataframe as a columnar Parquet file
import pandas as pd

df = pd.DataFrame({"user_id": [1, 2, 3], "score": [98.5, 92.1, 88.0]})
df.to_parquet("s3://my-datalake/raw/users.parquet", compression="snappy")
```
